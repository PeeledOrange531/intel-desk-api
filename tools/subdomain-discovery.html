Subdomain Discovery

Searches SSL certificate transparency logs via crt.sh · Passive reconnaissance — no direct scanning of target
How This Works
Every SSL/TLS certificate issued by a Certificate Authority must be publicly logged in a Certificate Transparency (CT) log — this is mandated by RFC 6962 and enforced by all major browsers. These logs reveal every subdomain a certificate has ever been issued for, including internal tools, staging environments, admin panels and API endpoints.

This tool queries crt.sh, the largest public CT log aggregator, and returns all matching subdomains deduplicated and tagged. Because this is a passive lookup of public records, no traffic is sent to the target domain. The target organisation cannot detect this search.
